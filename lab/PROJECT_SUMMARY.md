## 1. 기존 JINBUBU AI 마켓 프로젝트 현황

### 🎯 프로젝트 개요
- **목적**: AI 프롬프트 마켓플레이스 + 회원제 커뮤니티
- **타겟**: AI를 활용해 생산성을 높이고 싶은 크리에이터
- **핵심 가치**: 검증된 AI 프롬프트로 2시간 작업을 30분으로 단축

### ✅ 완료된 작업들

  - 통계: 75% 제작시간 단축, 400% 참여율 증가, 95% 일관성 달성
  - CTA: "💡 프롬프트 둘러보기" + "유튜브 채널"
  - **레이아웃**: 모든 요소 가로 배치 (제목, 서브타이틀, 통계, 버튼)

  4. 💡 **프롬프트 마켓** → /prompts.html
- **레이아웃**: 가로 배치 (flex-wrap: nowrap), 각 카드 280px 고정 너비
- **기존 중복 섹션**: `display: none` 처리 (소모임 배너, 프롬프트 바로가기)

#### 1.3 프롬프트 카탈로그
- **카드 크기**: 50% 축소 완료
  - 최소 너비: 300px → 180px
  - 패딩: 1.5rem → 1rem
  - 간격: 2rem → 1rem
  - 제목 폰트: 1.25rem → 0.9rem
  - 버튼 텍스트: "자세히 보기" → "보기"
- **한 줄 표시**: 4개 → 6개 카드
- **설명**: 2줄 제한 (line-clamp)

#### 1.4 회원제 시스템
- **가격 배너**: "💎 회원제로 운영 | 회원가입 후 맞춤 프롬프트 제공"
- **회원가입 프로세스 (3단계)**:
  1. **회원가입**: 이메일 + 비밀번호 (약 1분 소요)
  2. **소모임 오픈카톡 참여**: 가입 후 즉시 초대

  3. **첫 오프라인 모임 참석**:
     - AI 활용 능력 진단 (100점 만점)
     - 맞춤 학습 로드맵 제공
     - 실전 프롬프트 교육
     - 특전: 1:1 컨설팅 15분, 프리미엄 프롬프트 3종, 네트워킹

## 2. 새로운 실험실 랜딩페이지 프로젝트

### 🧪 프로젝트 개요
- **브랜드**: 퍼널 아키텍트 AI
- **슬로건**: "실패를 데이터로 해독하는 실험실"
- **핵심 차별화**: 강의가 아닌 "참여형 실험실"
- **중심 콘텐츠**: 0104.mov 영상

### 🎯 브랜드 세계관
- **목표**: 동기부여/강의/이론이 아닌 **삶과 채널의 구조를 직접 바꾸는 실험** 제공
- **금지 어휘**: 배워보세요, 알려드립니다, 강의합니다
- **사용 권장 어휘**: 참여, 실험, 구조, 해독, 직접 바꾸다

### 📐 참고 레퍼런스
- **URL**: https://gyiqbvre.gensparkspace.com/
- **특징**: 마우스를 따라 움직이는 인터랙티브 효과, 고급 원페이지 랜딩

### 🎬 6가지 핵심 임무
1. ❌ 기존의 일반적인 강의형 설명 구조 제거
2. 🎥 업로드 영상 (0104.mov)을 중심 콘텐츠 허브로 설정
3. 🎯 이 영상이 사이트 구조의 메인 트리거 콘텐츠가 되도록 재설계
4. 🔗 모든 섹션이 영상이 던지는 문제의식에서 자연스럽게 이어지도록 구조 재설계
5. 🔄 방문자가 **보기 → 공감 → 질문 → 참여** 흐름으로 전환 구조 재설계 (CTA, 섹션 순서, 버튼 문구)
6. 🧪 사이트를 **강의 사이트가 아니라 참여형 실험실 랜딩페이지**로 재구축

### 🏗️ 홈페이지 전체 구조도

#### SECTION 1: TRIGGER ZONE (트리거 존)
- **역할**: 첫 화면 임팩트, 호기심 자극
- **구성**:
  - 슬로건: "당신의 실패는 데이터다"
  - 마우스 인터랙티브 배경 (파티클/그라데이션)
  - 스크롤 유도 아이콘

#### SECTION 2: VIDEO HUB (영상 허브)
- **역할**: 핵심 메시지 전달, 문제의식 제기
- **구성**:
  - 제목: "실험의 시작"
  - 0104.mov 중앙 전체화면 배치
  - 재생 시 배경 dim 처리
  - 자막 + 자동재생 옵션

#### SECTION 3: RESONANCE (공감 존)
- **역할**: 감정적 연결, 문제 공감
- **구성**:
  - 제목: "당신도 이 질문을 던진 적 있는가?"
  - 3가지 공감 포인트 카드
  - Hover 시 확대 애니메이션
  - 실패 사례 시각화

#### SECTION 4: STRUCTURE (구조 존)
- **역할**: 솔루션 제시, 차별화 요소
- **구성**:
  - 제목: "우리는 구조를 바꾼다"
  - Before/After 비교 슬라이더
  - 데이터 기반 변화 그래프
  - 인터랙티브 구조 다이어그램

#### SECTION 5: EXPERIMENT ACCESS (실험실 입장)
- **역할**: 참여 유도, 전환 트리거
- **구성**:
  - 제목: "실험실 입장"
  - 참여 CTA 중앙 버튼
  - 3단계 프로세스 시각화
  - 실시간 참여자 수 카운터

#### SECTION 6: COMMITMENT (참여 신청)
- **역할**: 최종 전환, 데이터 수집
- **구성**:
  - 제목: "실험 참여 신청"
  - 간단한 폼 (이메일 + 현재 상황)
  - 즉시 피드백 (자동 진단 시작)
  - 다음 단계 안내

### 💬 영상 아래 메인 카피

```
당신의 채널이 망한 이유는
콘텐츠가 나빠서가 아니다.

구조가 잘못 설계됐다.

───────────────────

7,429개의 실패 데이터를 해독했다.
공통점은 하나였다.

실패한 사람들은
구조를 바꾸지 않았다.

───────────────────

실패를 데이터로,
데이터를 구조로 바꿔라.
```

### 🎯 참여 버튼 CTA 5종

1. **실험 참여 신청하기**
2. **내 구조 해독 시작**
3. **실패 데이터 제출하고 구조 받기**
4. **지금 실험실 입장**
5. **무료 진단 → 구조 변경**

### 🚀 사이트 첫 화면 슬로건 5종

1. **당신의 실패는 데이터다**
2. **구조를 바꾸지 않으면 결과도 바뀌지 않는다**
3. **7,429개 실패를 해독한 실험실**
4. **실패의 원인은 당신이 아니라 구조였다**
5. **실패 없이 변화 없다**

### 🔄 사용자 여정 흐름

```
1단계: 보기 (SEE)
├── 트리거 슬로건 충격
├── 스크롤 유도
└── 영상 시청

2단계: 공감 (FEEL)
├── 영상 후 공감 메시지
├── 실패 사례 3가지
└── "나도 그랬어" 순간

3단계: 질문 (QUESTION)
├── "왜 실패했을까?"
├── Before/After 비교
└── 구조적 문제 시각화

4단계: 참여 (ENGAGE)
├── CTA 버튼 클릭
├── 이메일 + 현재 상황 입력
└── 즉시 진단 시작
```

- **타이포그래피**: 굵은 헤드라인, 간결한 바디 텍스트
- **애니메이션**: Smooth scroll, Fade-in on scroll, Hover effects

### 🛠️ 기술 스택 (예상)
- **HTML5**: 시맨틱 마크업
- **CSS3**: Grid, Flexbox, CSS Variables, Animations
- **JavaScript**: Vanilla JS (마우스 인터랙션, 스크롤 애니메이션)
